Intelligent Routing

EPP -- Endpoint Picker

Every request is scored and routed by the EPP using a load-aware prefix-cache strategy. It considers cache affinity, current load, queue depth, and pod health.

Scoring Weights

Cache Affinity0.40
Load Balance0.25
Queue Depth0.20
Health Factor0.15

Strategy

load-aware-prefix-cache -- Routes requests to the pod most likely to already hold matching KV cache prefixes, weighted by current load.
